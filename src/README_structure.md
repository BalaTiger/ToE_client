# 前端结构说明

当前 `src` 目录按“入口层 / 常量层 / 游戏逻辑层 / 资源层”拆分。核心目标是让 `App.jsx` 保持对全局状态和界面编排的掌控，同时把可复用、可测试、变化频率不同的逻辑逐步抽离出去。

## 目录结构

```text
src/
├─ App.jsx
├─ App.css
├─ index.css
├─ main.jsx
├─ README_structure.md
├─ assets/
│  └─ react.svg
├─ constants/
│  └─ card.js
└─ game/
   ├─ ai.js
   ├─ coreUtils.js
   └─ viewerState.js
```

## 文件职责

### `App.jsx`

应用主入口，也是当前游戏前端的编排中心，主要负责：

- React 状态、`useEffect`、动画队列、阶段切换
- 玩家操作与 AI 回合的衔接
- 各类弹窗、面板、卡牌展示和教程 UI
- 将 `constants/` 与 `game/` 中的纯逻辑拼装成完整对局流程

原则上，`App.jsx` 更适合保留“流程编排”和“UI 决策”，不适合继续堆积纯工具逻辑。

### `constants/card.js`

静态卡牌与身份常量定义，包含：

- 区域牌正反面配置
- 邪神牌配置
- 角色、颜色、文案等基础常量

这类文件应保持“纯数据源”属性，尽量不放运行时逻辑。

### `game/coreUtils.js`

本轮新增的核心工具模块，承接通用规则工具与状态辅助逻辑，当前包括：

- 洗牌、数值裁剪、玩家深拷贝
- 区域牌 / 空白区域牌判断
- 正负面区域牌判定
- 牌面日志文本、追捕匹配判断
- 牌堆与身份分配
- 胜利手牌判定
- 空白区域牌回收、死亡清算、弃牌堆移除
- 带“两人一绳”联动的 HP 伤害处理
- 存活角色顺序计算

适合继续往这里放的内容：

- 不依赖 React 的规则工具
- 可被 AI、玩家流程、动画收尾共同复用的纯函数

### `game/ai.js`

本轮新增的 AI 逻辑模块，承接与“AI 决策”强相关的判断，当前包括：

- 邻座 / 存活目标计算
- 区域牌风险、收益、压力分类
- 不同阵营的区域牌评分
- `先到先得` 选牌
- 亮牌、夺牌、追捕继续与否等策略判断

适合继续往这里放的内容：

- 所有“AI 该不该这样做”的启发式逻辑
- 与 UI 无关、只依赖游戏状态的目标选择函数

### `game/viewerState.js`

本轮新增的多人视角旋转模块，负责：

- 将规范座位顺序旋转成“本地玩家视角”
- 反向还原旋转后的状态
- 维护 `abilityData / drawReveal / gameOver` 等座位索引字段的同步旋转

这部分逻辑独立后，多人模式相关改动不必再在 `App.jsx` 顶部维护大段旋转辅助函数。

### `main.jsx`

React 挂载入口，负责把应用挂到页面根节点。

### `App.css` / `index.css`

全局样式与应用级样式入口。当前项目的大量动态样式仍写在 `App.jsx` 内联对象和内联 `<style>` 中，后续如果继续做样式拆分，可以从这里开始整理。

## 当前拆分原则

这次拆分优先处理了“高复用、低 UI 耦合”的逻辑：

1. 规则工具与状态辅助逻辑移到 `game/coreUtils.js`
2. AI 目标选择与评分逻辑移到 `game/ai.js`
3. 多人视角旋转逻辑移到 `game/viewerState.js`

这样做的收益是：

- `App.jsx` 顶部不再堆积大量纯函数
- 规则逻辑和 UI 编排边界更清晰
- 后续做 bug 修复时，更容易定位“是规则问题、AI 问题还是视角映射问题”

## 下一步可继续拆分的方向

如果后续还要继续减轻 `App.jsx` 体积，优先级较高的方向是：

1. 把“胜负判定 / 回合推进 / 摸牌结算”再抽成 `game/flow` 类模块
2. 把“卡牌动效、死亡演出、转移动画描述”抽成 `game/anim` 或 `ui/anim` 模块
3. 把大型弹窗和教程 UI 拆成独立组件文件

目前这三块都还有拆分空间，但它们和现有状态编排耦合更深，适合在保持功能稳定的前提下分批推进，而不是一次性硬拆。
